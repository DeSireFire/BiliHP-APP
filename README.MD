## 模拟类型说明

软件有三种模拟模式，根据可实现的情况选择，客户端不可修改或选择：
 - 全模拟：最强模拟方式，从每条的Header-orign-refer到POST-GET值，均和官方接口一致
 - 部分模拟：次级模拟方式，本方式能模拟80%左右的内容，部分内容因为太麻烦所以没有完全模拟
 - 静态模拟：POST内容相同，但Header内容较为一致（本模拟方案和Github上其他项目的模拟方案一样）
 - 无模拟：接近CURL方案，死ぬかもしれません

## 礼物功能说明
+ 隐私（关注）：
    + 程序里面木有接入关注接口，也就是说软件不会主动对你的关注列表进行增加和删除操作（参考隐私接口）
    + 软件没有控制关注列表的接口，这种免费软件的流氓操作，我们都没有
    + 如果你莫名其妙关注某个主播，请关闭《天选时刻》功能，有些主播设定了必须关注后才能领奖，所以开启本功能后将会莫名其妙的关注一些有开天选时刻抽奖的头部主播
    + 重要：请务必先获取关注列表，然后锁定，接下来才能使用“一键取关”否则如果直接点击一键取关，
    那么你的关注列表就空了，因为程序会认为你没有关注任何人，全部取消关注

+ 隐私（礼物）：
    + 因为程序需要默认一个赠送直播间，未设定可能会出BUG，所以APP预设了赠送直播间为386281，
    大家记得先到“APP设定”中修改后再使用自动赠送和手动赠送功能哦

# C2C超级节点需求
+ 超级节点机器需要外网IP（开启超级节点后启动会检测，如果检测无法通过节点将降级成普通节点运行）
+ Windows版本需要修改注册表，增加MaxUserPort配置，允许软件使用5000-65535端口
+ Linux设备需要内存大于1G（夜间直播间数量达到2w的时候，内存太小会导致程序直接耗尽内存退出）
+ Windows设备内存需要达到3G（恩Windows内存占用就是会更多的，内存不足也会炸）
+ 超级节点需要申请，无法直接使用
+ 超级节点需要开放并允许UDP连入20202端口，如果关闭本端口，超级节点将会使用“单节点”模式运行而不会进化到“多播”模式

# C2C超级节点技术
+ BroadCast-UDP技术，就近连接（打洞技术在研究后上线）
+ 超级节点将纳入BiliHP矿机统计，越快向主网反馈有效弹幕，超级节点将会获得最高的积分，积分采用二分法获得分数，例如一个弹幕积分为10，那么第一个是5，第二个是2.5，第三个是1.25，以此类推
+ 超级节点对外提供WS-BroadCast与HTTP-API礼物数据
